Code Log 2: CSS Measuring Units

Absolute vs Relative

Absolute Length Units are fixed and don't change based on the screen size, parent elements, or user settings. They represent a specific physical or fixed size.

Relative Length Unites change dependinng on something else like the parent element, the root element, or the viewport size.

Describing Units

px (Pixels)

  • Abosolute Unit
  • Represents one unit of screen display resolution
  • Not always a physical screen pixel
  • Easy to understand
  • Doesn't automatically scale with user font preferences

pt (Points)

  • Abosolute Unit
  • 1pt = 1/72 of an inch
  • Mostly used in print design
  • Based on physical measurement
  • Not ideal for responsive web design or screen based layouts

cap

  • Relative Unit
  • Relative to the height of capital letters in the font
  • Specifically based on the font's cap height
  • Good for aligning elements visually with capital letters
  • More precise typography control

ex

  • Relative Unit
  • Relative to the x-height of the font
  • X-height = height of lowercase "x"
  • The x-height varies a lot between fonts, so ex can be unpredictable
  • More precise typography control

ch

  • Relative Unit
  • Based on the width of the "0" (zero) character
  • Good for controlling the width of the text
  • Great for readable text blocks
  • Helps control line length (ideal reading width is about 45-75 characters)

in (Inches)

  • Abosolute Unit
  • 1in = 96px in CSS
  • Best for print layouts and physical size control
  • Not ideal for responsive web pages

pc (Picas)

  • Abosolute Unit
  • 1pc = 12pt
  • 1pc - 1/6 of an inch
  • Used mostly in publishing and print design
  • Use for professional typography or print-based documents
